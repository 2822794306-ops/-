# 第二阶段 MVP 最小可行产品
第一版仅开发6个页面，完整作业批改闭环

## 页面1：老师后台
功能：创建作业

## 页面2：上传标准答案
支持文件类型：PDF、Word、图片

## 页面3：生成上传链接
示例链接：shturl.cc/smcXaUAfEJjhIzKQzYSM

## 页面4：家长上传页面
表单内容：
1. 学生姓名
2. 上传图片
3. 提交按钮

## 页面5：老师查看提交作业
1. 首页展示提交进度（例：32/35）
2. 点击学生查看详情：得分、对错标记、AI批改原因

## 页面6：统计页面
展示数据：
- 平均分
- 错误率
- 错题排行

# 第三阶段（技术选型）
## 前端 Frontend
- 框架：Next.js
- 样式：TailwindCSS

## 后端 Backend
- 框架：FastAPI

## 数据库 Database
- 第一版：SQLite（满足初期需求）
- 后期扩容替换：Postgres

## AI 大模型（批改能力）
可选调用：
- OpenAI
- DeepSeek
- Claude

## OCR 文字识别（作业图片解析）
- 方案：PaddleOCR

## 文件存储 Storage（作业图片、答案文件）
- 存储服务：Cloudflare R2
用途：存放所有上传图片、文档资源
## 业务流程
1. 老师后台创建作业 → 上传标准答案 → 生成专属上传链接
2. 家长打开链接，填写姓名、上传作业图片并提交
3. AI自动批改作业
4. 老师查看每位学生批改详情，查看全班统计数据

# 第四阶段（数据库设计）
项目一共6张数据表
## 1. Users 老师表
字段：代码、老师

## 2. Homework 作业表
字段：代码、作业标题、标准答案、截止时间

## 3. Students 学生表
字段：代码、学生姓名

## 4. Submission 提交记录表
字段：代码、关联学生、关联作业、上传图片地址、提交时间

## 5. Result AI批改结果表
字段：代码、关联提交记录、得分、批改详情JSON

## 6. Feedback 老师反馈表
字段：代码、关联批改结果、老师评语反馈

# 第五阶段（AI 四大提示词 Prompt）
项目仅使用4套核心Prompt，各司其职
## Prompt1：识别答案
输入：OCR识别文本
输出：结构化JSON标准答案数据

## Prompt2：AI批改
输入：标准答案、学生作答内容
输出：批改结果JSON（对错、得分、错题详情）

## Prompt3：成绩统计分析
输入：全班所有学生批改成绩
输出：班级学情分析（平均分、错误率、高频错题）

## Prompt4：课后家长反馈
输入：单名学生批改详情
输出：给家长可读的学情反馈文案

# 第六阶段（Claude 分任务开发规范）
## 错误方式
一次性指令：帮我写整个网站
问题：需求体量过大，AI极易逻辑混乱、代码翻车

## 正确开发方式
拆分最小粒度Task，单次只交付一个小功能，每日完成3~5个Task
### 示例任务拆分
Task001：项目初始化（前后端基础工程搭建）
Task002：开发老师首页页面
Task003：开发上传标准答案功能

## 开发原则
1. 不一次性要求完整全站代码，按单功能拆分任务
2. 单个Task完成后再下发下一个任务，分步迭代
3. 每个功能交付后核验无误，再推进下一模块
